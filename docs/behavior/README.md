# Поведение на бота — синтез от обратната връзка на клиента

Тук систематизираме обратната връзка от клиента (двата файла Feedback) в **приложими правила**. Това е „слоят за поведение", който досега не беше разработван — досегашният тест проверяваше само ориентацията в сайтмаповете.

Източници:
- `Feedback - Chatbot Prototype (1).docx` (наричан **fb1**)
- `Feedback 2 - Chatbot Prototype (1).docx` (наричан **fb2**)
- Имейл отговор до клиента (Филип), който поема посоката за надграждане.

## Файлове

| Файл | Съдържание |
|------|------------|
| [general-behavior.md](./general-behavior.md) | Общи правила за поведение → влизат в **основния промпт** (профил-първо, стил на отговора, линкове, темпо, без повторения) |
| [source-hierarchy-and-terminology.md](./source-hierarchy-and-terminology.md) | Йерархия на източниците по теми, остарели източници, терминология (закрила vs убежище), речник на съкращенията, конкретни ресурси за добавяне |
| [ui-and-product-notes.md](./ui-and-product-notes.md) | Начална страница, меню, езици, кликаеми подтеми, самообучение, свободно търсене в мрежата |

## Къде се прилага кое (за да няма конфликти)

| Слой | Какво поема от обратната връзка |
|------|-------------------------------|
| **Основен промпт** | Профил-първо, стил/формат, темпо, линкове като подкрепа, терминология, без „според ресурсите" всеки път, без повторения |
| **Йерархия на източниците** (конфиг/логика) | Кои източници първо по тема; кои да се избягват (ukraine.gov.bg) |
| **Сценарии** (per-topic) | Конкретните стъпки, организации, ресурси за темата |
| **UI/продукт** | Начална страница, меню, кликаеми подтеми, езици |

## Едно изречение, което обединява цялата обратна връзка

> Ботът не трябва да изсипва линкове и условни варианти. Първо изяснява личната ситуация (откъде е, какъв статут има/търси, възраст, локация, работи ли…), после дава **директен, конкретен отговор в самия текст** (с конкретни места/организации/сайтове), а линковете са само „виж повече".
